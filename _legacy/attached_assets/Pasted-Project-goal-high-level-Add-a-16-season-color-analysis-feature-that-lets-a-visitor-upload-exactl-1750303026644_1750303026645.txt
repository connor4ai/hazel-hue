Project goal (high level)
Add a “16-season color analysis” feature that lets a visitor upload exactly three selfies and then:
Gets their season, palette, star colors, etc., from OpenAI GPT-4o (vision).
Shows the results in a swipeable card UI.
Generates a CMYK-ready PDF of the palette.
Provides a shareable link to the same report.
Constraints
• Do not change or delete any existing files or logic—only add what’s necessary and register it.
• Match the current tech stack you see in the repo (Express/TypeScript on the server, React + Vite + Tailwind on the client).
• Keep the new feature isolated (own route, own React page, own folders).
Back-end tasks
1. Install any extra NPM packages you need—expect to need OpenAI SDK, a file-upload helper, a small color library for CMYK conversion, and a simple PDF generator.
2. Create a new Express router under server/routes called “colorAnalysis”.
  • Endpoint: POST /api/analyze
  • Accept exactly three image files (“photos”).
  • Send those images plus a fixed prompt (the one that returns season, palette JSON blocks, etc.) to the GPT-4o model.
  • Parse the model’s markdown to pull out the full palette array and the star-color array.
  • Generate a CMYK PDF of all palette swatches and save it to uploads/.
  • Return JSON: { id, markdown, palette, wowColors }.
3. Add GET /pdf/:id (streams the saved PDF) and GET /share/:id (returns the same markdown) to the same router.
4. Register the router in the main Express app without disturbing existing routes.
5. Require an OPENAI_API_KEY environment variable (create the key placeholder in the Secrets panel).
Front-end tasks
1. Install Swiper (or any lightweight swipe library) for a card-style interface.
2. Add a new React page at /color-analysis that:
  • Lets the user pick exactly three image files.
  • Calls POST /api/analyze and waits for the JSON response.
  • Displays three swipeable slides:
     – Slide 1: season badge line, pulled from the markdown’s first line.
     – Slide 2: grid of all palette colors.
     – Slide 3: grid of star (“wow”) colors with a bold border.
  • Shows two buttons under the swiper:
     “Download PDF” → opens /pdf/:id in a new tab.
     “Share” → copies /share/:id to clipboard and alerts “Link copied!”.
3. Wire the route into the existing client-side router so visiting /color-analysis renders the new page.
4. Use the existing Tailwind setup for basic spacing, rounded corners, and shadows; no extra styling library needed.
After implementing
• Run the dev server, navigate to /color-analysis, and verify that uploading three test photos returns the swiper UI, creates a PDF download, and copies a share link.
• Echo back “🎨 Color-analysis feature integrated and working” when everything passes manual smoke-test.
